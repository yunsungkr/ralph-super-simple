# ralph-super-simple

This repository describes an **iterative execution pattern** inspired by the ralph idea rather than a concrete agent implementation.

It uses [Claude CLI](https://claude.com/claude-code), [OMC (oh-my-claudecode)](https://github.com/nicobailey-omc/oh-my-claudecode), and [ralph-claude-code](https://github.com/frankbria/ralph-claude-code) for skill execution, and [Codex CLI](https://github.com/openai/codex) with [Gemini CLI](https://github.com/google-gemini/gemini-cli) for iterative discussions.

## Core Pattern

```python
for phase in phases:
    prompt = read(f"{phase}/PROMPT.md")
    while "EXIT_SIGNAL: true" not in run(f"claude -p '{prompt}'"):
        prompt = "Check fix_plan.md and perform the next task."
    run(f"bash {phase}/verify.sh")
```

This is the entire pattern: split the plan into phase-specific files, then run each phase repeatedly in an independent session. Each session aims to implement one phase.

**Phase-based planning is the key.** Create a plan that defines the structure, exit conditions, and verification criteria for each phase, then implement it with the provided skills. Skills such as `/ralplan` from oh-my-claudecode work well for generating this kind of phase plan.

## Skills

| Skill | Purpose | Required Tools |
| --- | --- | --- |
| **ralphss-import** | Runs a plan in an independent iterative `claude -p` loop | Claude CLI |
| **ralphcc-import** | Runs a plan iteratively with the `ralph-claude-code` agent | Claude CLI + ralph-claude-code |
| **ralph-discussion** | Runs alternating multi-model discussions on a specific topic | Codex CLI + Gemini CLI |

### ralphss-import — Independent `claude -p` Iteration

Organizes an implementation plan into a `.ralphss/` directory structure. This skill runs through `claude -p` by default, but you can edit it to replace the primary CLI with another one such as Codex or Gemini.

```bash
/ralphss-import my-plan.md     # Create .ralphss/
bash .ralphss/run.sh           # Run in a separate terminal
rm -rf .ralphss/               # Clean up
```

**Generated by `/ralphss-import`:**

```text
.ralphss/
├── run.sh                     # Autonomous loop (invokes `claude -p` directly)
├── MASTER_PLAN.md             # Overall plan
├── AGENT.md                   # Build/test commands
├── specs/requirements.md      # Detailed requirements
└── phases/
    ├── phase-1/
    │   ├── PROMPT.md          # Loop control instructions
    │   ├── fix_plan.md        # Commit-sized task checklist
    │   └── verify.sh          # Phase-specific verification script
    └── ...
```

### ralphcc-import — `ralph-claude-code` Integration

Organizes an implementation plan into a `.ralph/` directory structure. It uses `ralph-claude-code`'s `--resume` session management and `EXIT_SIGNAL` detection.

```bash
/ralphcc-import                # Create .ralph/
bash .ralph/run.sh             # Run in a separate terminal with automatic phase switching
/ralphcc-clear                 # Clean up ralph-claude-code generated files
```

**Generated by `/ralphcc-import`:**

```text
.ralph/
├── run.sh                     # Phase switching + ralph invocation
├── MASTER_PLAN.md             # Overall plan
├── AGENT.md                   # Build/test commands
├── specs/requirements.md      # Detailed requirements
└── phases/
    ├── phase-1/
    │   ├── PROMPT.md
    │   ├── fix_plan.md
    │   └── verify.sh
    └── ...
```

### ralph-discussion — Multi-Model Discussion

Codex and Gemini discuss a specific topic in alternating rounds, usually over 3 to 5 rounds.

```bash
/ralph-discussion my-topic.md  # Create .discussion/
bash .discussion/run.sh        # Run alternating Codex ↔ Gemini rounds
```

**Generated by `/ralph-discussion`:**

```text
.discussion/
├── run.sh                     # Round runner
├── topic.md                   # Copy of the original topic
├── synthesis.md               # Copy of the last round's output (generated at runtime)
└── rounds/
    ├── round-1-codex/
    │   ├── instruction.md     # Round instructions (created during import)
    │   ├── prompt.md          # Actual prompt (generated at runtime)
    │   └── output.md          # Response (generated at runtime)
    ├── round-2-gemini/
    │   └── ...
    ├── round-3-codex/
    │   └── ...
    └── ...
```

## Terminology

| Term | Description |
| --- | --- |
| `PROMPT.md` | Loop-control instructions for a phase. Keep it lightweight, and move detailed specs into `specs/`. |
| `fix_plan.md` | A commit-sized task checklist. Best practice: 1 task = 1 commit, with changes limited to 3 to 5 files. |
| `verify.sh` | A script that verifies phase completion using concrete commands such as `grep`, `pytest`, or `curl`. |
| `EXIT_SIGNAL` | The loop termination signal, derived from [ralph-claude-code](https://github.com/frankbria/ralph-claude-code). |
| `MASTER_PLAN.md` | The overall multi-phase plan. |
| `AGENT.md` | Defines build and test commands. |
| `run.sh` | The integrated script that handles the autonomous loop and phase switching. |
| `instruction.md` | Defines the specific goal for a discussion round. |
| `synthesis.md` | The final synthesized result of a discussion. |
